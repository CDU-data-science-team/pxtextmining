pxtextmining.helpers.ordinal_classification module¶

class pxtextmining.helpers.ordinal_classification.OrdinalClassifier(estimator=LogisticRegression(), clfs={}, y_factorized=None, unique_class=None, class_dict=None, theme=None, target_class_value='3', theme_class_value=1)¶

Bases: sklearn.base.BaseEstimator

Estimator class for building an ordinal classification model using the method of Frank and Hall (2001) The code in this class is based on code published online in this post.

NOTE: As described later, argument theme is for internal use by Nottinghamshire Healthcare NHS Foundation Trust or other trusts who use the theme (“Access”, “Environment/ facilities” etc.) labels. It can otherwise be safely ignored.

Parameters

  • estimator – A Scikit-learn classifier.

  • clfs (dict) – Helper variable. Defined inside the class.

  • y_factorized – Helper variable. Defined inside the class.

  • unique_class – Helper variable. Defined inside the class.

  • class_dict (dict) – Helper variable. Defined inside the class.

  • theme (str) – For internal use by Nottinghamshire Healthcare NHS Foundation Trust or other trusts that use theme labels (“Access”, “Environment/ facilities” etc.). The column name of the theme variable. Defaults to None. If supplied, the theme variable will be used as a predictor (along with the text predictor) in the model that is fitted with criticality as the response variable. The rationale is two-fold. First, to help the model improve predictions on criticality when the theme labels are readily available. Second, to force the criticality for “Couldn’t be improved” to always be “3” in the training and test data, as well as in the predictions. This is the only criticality value that “Couldn’t be improved” can take, so by forcing it to always be “3”, we are improving model performance, but are also correcting possible erroneous assignments of values other than “3” that are attributed to human error.

  • target_class_value (str) – The criticality value to assign to “Couldn’t be improved”.

  • theme_class_value (int) – The value of “Couldn’t be improved” in the transformed (e.g. one-hot encoded) theme column.

pxtextmining.helpers.oversampling.random_over_sampler_dictionary(y, threshold=200, up_balancing_counts=300)¶

Function that detects rare classes.

Finds classes with counts fewer than a specified threshold. The function performs a few validity checks:

1. The threshold must be smaller than the up-balancing number(s). When it is not, the latter takes the value of the former.

2. When the up-balancing number is zero or the threshold is smaller than all class counts, then the function returns the original counts.

The validity checks ensure that the function does not stop the script. It is completely the user’s responsibility to ensure that the supplied values are meaningful. For example, if each of the rare classes are > 200 in number but the threshold were 100 in one run and 150 in another run of the pipeline, then the result would be the original counts in both cases, i.e. there would be a redundant repetition of runs. Finally, the up-balancing number can be 0, an integer or a list of integers with length = number of rare classes. It is the user’s responsibility to ensure that, when it is a list, it has the correct length.

Parameters

  • y (ndarray) – The dependent variable. Shape (n_samples, ).

  • threshold (int) – The class count below which a class is considered rare.

  • up_balancing_counts (array[int]) – The number by which to up-balance a class.

Returns

rare_classes (dict): Keys are the rare classes and values are the user-specified up-balancing numbers for each class.
